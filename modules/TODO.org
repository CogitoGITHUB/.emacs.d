* Yasnippet nested/mirroring features (concise)

Nested snippets: expand a snippet from inside another so inner snippet has independent tabstops.
Mirrors: reuse a field elsewhere using ${n} so updating one spot updates all mirrors automatically.
Transforms: apply elisp/text transforms to a field on insertion (e.g., lowercase, regexp replace).
Conditional fields: evaluate elisp to choose content or whether to include a block.
Dynamic/snippet expansion: run elisp in a snippet to compute or expand inner snippets programmatically.
Tabstop lists & ordering: arbitrary tabstop indices, jump order, and $0 final cursor position.
Choice fields: provide alternatives within a field (${1|opt1,opt2|}) for quick selection.
Regex/format helpers: format or sanitize field input before insertion via evaluated expressions.
Recursive generation: programmatically generate repeated sub-blocks (e.g., list of N subtasks) during expansion.
Hooks & post-expand actions: code run after expansion to reposition point, run mode hooks, or trigger further completion.

* Goals
- Make god-mode a momentary, high-priority transient layer for single-key commands.
- Preserve meow’s modal semantics (normal/insert/visual) and general’s leader/keybinding helpers.
- Minimize conflicts and cognitive load.

* Key concepts & patterns
** 1. Role separation
- Meow :: global modal editing (normal/insert/visual) and motion/editing primitives.
- General :: centralized place to declare leader keys, prefixes, and contextual bindings.
- God-mode :: temporary “modifier” layer intended for fast, single-key commands (command palette, window/buffer management, quick toggles).

** 2. Activation model
- Momentary (hold-to-use) — preferred: press and hold a specific physical key (or mapped key) to enter god-mode; releasing returns to previous state.
- Latch/toggle — alternative: press once to enter and again to exit; useful if hardware/terminal cannot detect key releases.
- Leader-triggered — hybrid: use a leader (via general) to open a short-lived god-mode-like map for a few seconds.

** 3. Scope & granularity
- Global vs buffer-local :: Prefer a small global god keymap for universal actions, allow buffer-local god maps for mode-specific actions.
- Minimal keyset :: Limit god-mode to a curated set of very frequently used single-letter keys.

** 4. Priority & interception
- Temporarily suspend/disable meow while god-mode active.
- Or use an overriding keymap mechanism so god-mode wins without toggling meow state.

** 5. Key choice & ergonomics
- Choose a physical key that’s easy to hold (space, remapped caps-lock, unused function key).
- Consider long-press or chord if you already use space as leader.
- Ensure activation key works in terminal vs GUI.

** 6. Command selection & design
- Favor single-keystroke commands: window splits, window switching, buffer switching, file open, save, kill buffer, eval buffer, toggles, compile/run.
- Mnemonic mapping examples: w = window, b = buffer, f = file, s = save, q = quit.

** 7. Conflict mitigation
- Keep god-mode small to avoid overlap with meow motions.
- Use short timeout for latch/hybrid modes.
- Centralize mappings using general to audit overlaps.

** 8. Discoverability & fallback
- Show transient hint (echo area or posframe) listing available single-key commands when god-mode is entered.
- If key is unmapped in god-mode, fall back to meow behavior or exit god-mode.

** 9. Integration with leader keys
- Use general to define god-mode activation via leader if preferred.
- Keep leader multi-key, god-mode single-key; avoid duplicating responsibilities.

** 10. Testing strategy
- Start with 6–10 key pilot set for a week; evaluate usage and adjust.
- Track accidental activations and collisions.

** 11. Environment considerations
- GUI :: can use key-release detection for true hold behavior.
- Terminal/remote :: key-release may not be available — prefer latch or leader-triggered modes.

** 12. Recovery & safety
- Provide a global escape (e.g., ESC) that exits god-mode and restores meow.
- Ensure essential typing keys are never permanently blocked.

* Example workflows (high-level)
- Hold Space :: press-and-hold Space to enter god-mode; while held press f/b/s/w/q; release to return.
- Leader → God :: press SPC g to open a 3s god-mode where single keys perform window and buffer ops, auto-exit after inactivity.
- Toggle for heavy tasks :: press CapsLock to enter persistent god-mode, press again to exit.

* Checklist before enabling
- Pick activation key that works in your setup.
- Define 8–12 prioritized single-key commands.
- Decide momentary vs latch based on environment.
- Plan conflict resolution (suspend meow or use overriding map).
- Add visible hint and an escape key.
- Pilot and iterate.
